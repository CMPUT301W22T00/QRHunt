Coding Conventions

General
-Lines limited to 100 characters in length
-Multi-line comments use /* ... */ and not // ... // ...
-End brackets must be their own line and match indentation of the declaration
-Use correct indentation
-Function definitions and body should always be on separate lines


Names
-in xml, ids use snake case
    - ids are named with a where_what convention with an additional name for its type (e.g., qr_profile_score or qr_profile_score_text)
-function and method names use camel case
-Constructors use pascal case
-Variable and object names use camel case
